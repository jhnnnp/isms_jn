개인정보 탐지 및 비식별화 시연

텍스트·파일에서 개인정보를 탐지하고, 마스킹·암호화·복호화 결과를 한 화면에서 확인합니다.

  1. 동작 선택 후 입력·실행
  2. 쉬운 요약과 JSON으로 검증
  3. 필요 시 복호화 탭

샘플 입력이 준비되어 있습니다. 이용 안내에서 흐름을 보고, 텍스트 처리에서 API 실행하기 — 비식별화 후 복호화 탭으로 이어집니다.

텍스트 처리

POST /scan/text

비식별화를 고르면 “어떤 종류를 잠글지” 칩과 비밀키 입력란이 아래에 나타납니다.

칩으로 고른 종류만 잠금 문자열(ENC)로 바뀌고, 고르지 않은 종류는 가리기(마스킹)만 됩니다.

암호화와 복호화에 같은 값을 써야 합니다. 데모는 64개의 a가 기본입니다. (형식은 서버에서 맞춥니다.)

데모는 영구 저장소에 입력·키를 기록하지 않습니다. 운영 환경에서는 HTTPS와 KMS 등으로 키를 분리하는 것이 일반적입니다.

선택한 엔드포인트로 요청을 보냅니다.

업로드 후 multipart로 전송됩니다.

복호화

POST /decrypt/text

권장 시나리오 (한 바퀴)

  1. 텍스트 처리에서 “비식별화”를 고르고, 잠글 종류(주민·전화·이메일)를 선택한 뒤 API 실행합니다.
  2. 응답의 outputText와 비밀키가 이 탭 입력란에 자동으로 채워집니다. (직접 붙여 넣어도 됩니다.)
  3. 여기서 복호화 실행하면 ENC::종류::페이로드만 원문으로 바뀐 문장이 나옵니다. 마스킹만 된 글은 복구되지 않습니다.

이 API는 이 데모가 만든 ENC::... 문자열만 처리합니다 (AES-GCM). 일반 암호문·다른 포맷은 대상이 아닙니다.

아래 “시나리오 샘플 채우기”는 데모 키로 미리 만들어 둔 유효한 예시 문장입니다. 실행하면 주민·전화·이메일 세 토큰이 풀립니다.

비식별화에서 잠금에 썼던 키와 바이트 단위로 동일해야 합니다. (데모 기본 64개 a)

샘플은 키·문장이 이미 맞물려 있습니다. 바로 실행해 볼 수 있습니다.

이용 안내

가이드

이 데모의 흐름(간편 튜토리얼), 용어, 이 화면에서 하는 일을 한곳에 모았습니다. 실행은 텍스트·파일·복호화 탭에서 하시면 됩니다.

이 화면에서 해보는 일 (비전문가용)

  • 찾기: 입력한 글·파일에서 개인정보로 의심되는 부분이 어디인지 찾습니다.
  • 가리기·잠그기: 찾은 항목을 별표 처리처럼 가리거나, 선택한 항목만 잠금 문자열(ENC)로 바꿉니다.
  • 풀기: 비식별화에서 잠금(ENC)을 썼다면, 실행 후 복호화 탭에 결과가 채워지니 같은 키로 원문이 되돌아오는지 확인합니다. (복호화 탭에도 바로 시험할 수 있는 샘플이 있습니다.)

각 기능 실행 후 나오는 JSON은 시스템 연동·개발용 상세 데이터입니다. 어렵게 느껴지면 실행 결과 위쪽의 쉬운 요약만 보셔도 됩니다.

간편 튜토리얼

1

탐지 vs 비식별화

단순 탐지는 “어디에 무엇이 있는지”만 봅니다. 비식별화는 그다음으로 글자를 실제로 가리거나 잠급니다.

2

칩과 키

비식별화에서 칩으로 고른 종류만 ENC로 잠기고, 나머지는 마스킹됩니다. 비밀키는 비식별화와 복호화에 동일해야 합니다.

3

복호화 탭

텍스트 비식별화가 끝나면 복호화 탭에 결과와 키가 채워집니다. 그대로 실행해 원문이 돌아오는지 확인합니다.

개인정보 탐지
문장이나 파일 안에서 주민번호·전화·이메일처럼 보이는 부분을 어디에 있는지 찾아 목록으로 알려 주는 단계입니다. 아직 가리거나 바꾸지는 않습니다.
비식별화
찾은 개인정보를 읽기 어렵게 만드는 단계입니다. 이 데모에서는 가리기(마스킹)와 잠금(암호화)을 함께 씁니다.
마스킹
일부만 보이게 가리는 방식입니다. 예: 전화번호 가운데를 ****로 바꿉니다.
ENC / 잠금 문자열
ENC::... 형태로 바뀐 값은 원문을 직접 보여 주지 않습니다. 같은 비밀키로만 다시 풀 수 있습니다.
복호화
잠금(ENC)으로 바꾼 내용을, 암호화할 때 쓴 것과 같은 비밀키로 원문으로 되돌리는 단계입니다.

화면과 API 대응

ISMS-P 2.6(식별 결과 구조화)·3.2(마스킹·암호화 분리)에 맞춰 응답을 필드 단위로 제공합니다. 데모 화면과 실제 호출 경로는 아래와 같습니다.

텍스트 탐지 → POST /scan/text (JSON: text)
텍스트 비식별화 → POST /redact/text (JSON: text, encryptTypes, encryptionKey)
파일 탐지 → POST /scan/file (multipart: upload)
파일 비식별화 → POST /redact/file (multipart: upload, encryptTypes, encryptionKey)
복호화 → POST /decrypt/text (JSON: text, encryptionKey)
